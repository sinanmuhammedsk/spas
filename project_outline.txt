1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
The Student Performance Analysis System (SPAS) is a premium, professional web-
based analytics platform designed to predict academic outcomes using Machine 
Learning. By collecting and analyzing 18 core indicators across academic history, 
study behaviors, and socio-environmental factors, the system fits decision 
boundaries, projects final grades, identifies critical risk factors, and 
synthesizes personalized student advisory recommendations.

3. MACHINE LEARNING MODEL SPECIFICATIONS
--------------------------------------------------------------------------------
The backend is driven by a scikit-learn Machine Learning pipeline trained on
1,500 student records representing various grade brackets.

A. Core Algorithm:
   * RandomForestRegressor (ensemble of 100 decision trees)
   * Chosen for its non-linear relationship parsing, immunity to overfitting on 
     small tables, and ability to extract direct Gini feature importances.

B. Input Features (18 Parameters):
   * Academic Baselines (Numeric):
     - Previous Exam Score (0 - 100%)
     - Attendance Percentage (0 - 100%)
     - Assignments Completed (0 - 15 total)
     - Internal Assessment Score (0 - 30 total marks)
     - Subjects Failed Count (0 - 10 subjects)
   * Study Habits (Numeric & Categorical):
     - Daily Study Hours (0 - 12 hours)
     - Weekly Self-Study Hours (0 - 60 hours)
     - Tuition/Coaching Classes (Yes / No)
     - Exam Preparation Level (Low / Medium / High)
   * Demographics & Personal (Numeric & Categorical):
     - Student Name & ID (Identifiers)
     - Age (10 - 30 years)
     - Gender (Male / Female / Other)
     - Class / Grade Level (Grade 9 to Undergrad)
   * Environmental Factors (Categorical):
     - Internet Access at Home (Yes / No)
     - Extracurricular Activities (Yes / No)
     - Parental Education Level (High School / Bachelor / Master / PhD)
     - Family Academic Support (Low / Medium / High)

C. Model Training Metrics (Verified on 30% Holdout Test Partition):
   * R-squared (R2) Coefficient: 0.9147 (Explains 91.5% of score variance)
   * Mean Squared Error (MSE): 13.68
   * Mean Absolute Error (MAE): 2.91%

D. Model Feature Importances (Gini Index Split Contribution Weights):
   * 1. Subjects Failed Count      - 74.9% (Critical Risk Marker)
   * 2. Assignments Completed      -  6.8% (Self-Discipline Indicator)
   * 3. Attendance Percentage      -  4.3% (Engagement Driver)
   * 4. Exam Preparation Level     -  3.6% (Readiness Metric)
   * 5. Internal Assessment Marks  -  3.2% (Mid-term Standing)
   * 6. Previous Exam Score        -  2.2% (Historical Baseline)
   * 7. Cumulative Others          -  5.0% (Socio-environmental)


4. BACKEND WEB API ARCHITECTURE (FLASK)
--------------------------------------------------------------------------------
The server is structured using the Flask micro-framework and exposes three routes:

* Route 1: GET /
  - Serves 'index.html' directly as the entrypoint for the client browser.
* Route 2: GET /api/metrics
  - Returns global model R2, MSE, MAE, and Gini feature importances in JSON.
  - The UI reads these directly to generate live model charts on the fly.
* Route 3: POST /api/predict
  - Accepts a JSON object containing the student's 18 inputs.
  - Standardizes the categories using encoded mappings matching the trainer, 
    feeds the features to the model, classifies the outcome into categories 
    (Excellent/Good/Average/Poor), evaluates risk status (Low/Medium/High), 
    and synthesizes positive drivers, negative risks, and a personalized 
    action plan recommendations list.


5. FRONTEND USER INTERFACE SPECIFICATIONS
--------------------------------------------------------------------------------
The user interface is designed as an interactive, single-page dashboard shell 
divided into three main sections:

A. Theme Manager (Dark / Light Mode Toggle):
   * Saves preference locally to localStorage.
   * Dynamically alters global CSS variables and re-renders Chart.js grids 
     instantly to match contrast grids (light mode white/gray, dark mode navy).

B. Page 1: Student Information & Data Collection Form:
   * 18 validated inputs structured in a clean, glassmorphic card layout.
   * Real-time Data Quality indicator:
     - Features an animated progress ring that monitors form completion.
     - Performs range validations (e.g. Prev Score 0-100, Daily Study 0-12) 
       to block submittals of corrupted boundaries.
   * Features a Quick Demo Loader widget to automatically populate templates:
     - Star Student (High performance, low risk)
     - Average Student (Medium performance, medium risk)
     - At-Risk Student (Critical attendance, failing backlogs, high risk)

C. Prediction Loader overlay:
   * A full-screen backdrop that shows up on submission.
   * Simulates inference engine logs step-by-step:
     "Validating Input Data..." -> "Preparing Features..." -> 
     "Running Machine Learning Model..." -> "Generating Performance Analysis..." 
     -> "Prediction Complete".
   * Automatically falls back to a high-fidelity client-side mathematical 
     prediction heuristic if the Flask backend is unreachable (zero timeout locks).

D. Page 2: Predictive Analytics Dashboard (Analysis Hub):
   * Student Summary Profile Card showing input metrics.
   * Predicted Performance circular gauge matching the final grade percentage.
   * Academic At-Risk gauge assessing dropout/failure likelihood.
   * Dynamic lists identifying Positive Drivers vs Risk Elements.
   * Dynamic Academic Strengths vs Areas for Improvement list.
   * Four interactive Chart.js charts:
     - Attendance Bar Chart (Student vs Class average)
     - Study Hours Allocation (Daily and Weekly vs Suggested targets)
     - Score Projection (Previous vs Predicted score)
     - Feature Group Contribution (doughnut showing Academic vs Habit weights)
     - Performance Growth Line Chart (Class Test 1 -> CT-2 -> Internals -> Final)
   * AI-generated Action Recommendations mini-cards.

E. Page 3: Model Metrics & Diagnostic Insights:
   * Horizontal bar chart displaying real Gini importance indices from model.pkl.
   * Multi-Class ROC-AUC (Sensitivity vs Specificity) curve visualization.
   * Interactive Confusion Matrix (Actual vs Predicted classifications).
   * Tabular list showing statistical metrics (R2, MSE, MAE, split ratios).

F. Printable / PDF Report Generator:
   * Triggered via "Print" or "Download Report" buttons.
   * Employs custom CSS print queries (`@media print`) that hide sidebars and 
     buttons, layout-collapsing the page into a clean, double-page official 
     academic transcript document ready for counselor signatures.
